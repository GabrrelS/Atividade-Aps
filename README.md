Este repositório contém a resolução de exercícios da disciplina de Análise e Projeto de Sistemas 1, com foco na identificação de classes, atributos, métodos e relacionamentos a partir de cenários propostos.

Questão 1 – Conta de Luz

O que foi pedido:
Identificar classes, atributos e métodos para controle de gastos com conta de luz, incluindo consultas de maior e menor consumo.

Como foi resolvido:
Foi criada uma classe para representar a conta de luz com seus atributos principais e uma estrutura de controle para armazenar os dados e realizar as consultas de maior e menor consumo.

Questão 2 – Classe TextoSaída

O que foi pedido:
Modelar uma classe para configurar e exibir um texto com propriedades visuais e tipo de componente.

Como foi resolvido:
Foi criada uma classe com atributos de formatação e métodos para configurar e exibir o texto, utilizando valores controlados para cores e tipos de exibição.

Questão 3 – Boneco em Movimento

O que foi pedido:
Modelar uma classe que represente um boneco com posição e direção.

Como foi resolvido:
Foi criada uma classe com atributos de posição e direção, além de métodos para movimentação, alterando as coordenadas conforme a direção.

Questão 4 – Horário de Remédios

O que foi pedido:
Modelar um sistema de controle de horários de medicamentos com geração e reorganização de horários.

Como foi resolvido:
Foram criadas classes com atributos de controle do remédio e métodos para gerar horários, organizar a rotina diária e ajustar em caso de atraso.

Questão 5 – Gastos Diários

O que foi pedido:
Modelar um sistema de controle de gastos com agrupamento por tipo e forma de pagamento.

Como foi resolvido:
Foi criada uma classe de gasto e uma estrutura para agrupar os dados e calcular totais por categoria e forma de pagamento.

Questão 6 – Comanda Eletrônica

O que foi pedido:
Modelar um sistema de comanda com registro de produtos e cálculo do valor total.

Como foi resolvido:
Foram criadas classes para comanda, produto e itens, com métodos para adicionar registros e calcular o total da compra.

Questão 7 – Lista de Compras

O que foi pedido:
Modelar uma lista de compras com controle de quantidades e preços.

Como foi resolvido:
Foi criada uma classe de item com atributos principais e métodos para cálculo do total e atualização de valores.

Questão 8 – Coleção de CDs

O que foi pedido:
Modelar uma coleção de CDs com informações básicas.

Como foi resolvido:
Foi criada uma classe CD com atributos simples e métodos de cadastro e listagem.

Questão 9 – Coleção de CDs (Variação)

O que foi pedido:
Expandir o modelo para suportar múltiplos artistas, músicas e características do CD.

Como foi resolvido:
Foram adicionadas classes para artista e música, além de ajustes na classe CD para suportar múltiplas relações e consultas.

Questão 10 – Sala de Reunião

O que foi pedido:
Modelar um sistema de agendamento de salas com controle de disponibilidade e realocação.

Como foi resolvido:
Foram criadas classes para sala e reunião, com métodos para agendamento, alteração e consulta de horários.

Questão 11 – Herança

O que foi pedido:
Criar uma superclasse com atributos e métodos comuns entre Funcionário e Cliente.

Como foi resolvido:
Foi criada uma superclasse com os atributos compartilhados, e as classes específicas passaram a herdar dessa estrutura.
